# 掌握常用命令行工具

> 说实话，我当年学命令行，完全是被逼的——实验室的服务器没有图形界面，不会敲命令连代码都跑不了。

但用久了会发现，命令行这玩意儿真的比图形界面高效太多了。

## 文件操作——最常用的几个

### ls——查看目录内容

```bash
ls              # 简单列出
ls -l           # 详细信息
ls -a           # 包括隐藏文件
ls -lh          # 人性化显示大小
```

### cd——切换目录

```bash
cd ~            # 回家
cd -            # 去上一次待的目录
cd ..           # 上一级
```

### cp/mv/rm——复制/移动/删除

```bash
cp file1 file2         # 复制
cp -r dir1 dir2        # 复制目录
mv oldname newname     # 重命名/移动
rm file                # 删除
rm -rf dir             # 强制删除目录（慎用！）
```

> **血的教训**：rm -rf 一定要看清楚再按回车。我有个朋友，把 `rm -rf /tmp/*` 打成了 `rm -rf / *`——整个服务器当场去世。

### mkdir——创建目录

```bash
mkdir dirname
mkdir -p a/b/c        # 递归创建
```

## 查看文件内容

### cat——直接显示

```bash
cat file.txt
```

适合小文件。大文件会刷屏到你怀疑人生。

### less——分页查看

```bash
less file.txt
```

按 `q` 退出，`空格` 下一页，`b` 上一页，`/` 搜索。

看日志的时候这个最常用。

### head/tail——看头/看尾

```bash
head -n 20 file.txt     # 前 20 行
tail -f file.txt        # 实时刷新（看日志神器）
tail -n 100 file.txt    # 后 100 行
```

比赛现场调试的时候，`tail -f` 实时看日志是基本操作。

### wc——统计行数/字数

```bash
wc -l file.txt          # 行数
wc -w file.txt          # 单词数
wc -c file.txt          # 字符数
```

## 搜索——找文件/找内容

### find——找文件

```bash
find . -name "*.cpp"            # 按名字找
find . -type d                  # 找目录
find . -mtime -1                # 找最近一天修改的文件
find . -size +100M              # 找大于 100M 的文件
```

### grep——找内容

```bash
grep "error" file.txt           # 在文件中找 "error"
grep -r "error" ./src/         # 递归搜索
grep -n "error" file.txt       # 显示行号
grep -v "error" file.txt       # 反向匹配（不包含 error 的）
```

> 调试代码的时候，`grep -rn "error" ./` 这个命令能救你的命。

### locate——快速找文件

```bash
locate myfile
```

比 find 快，因为它搜索的是数据库。但需要先 `updatedb`。

## 管道和重定向——组合命令

管道 `|` 把前一个命令的输出交给后一个命令：
```bash
ls -l | grep ".cpp"             # 列出 cpp 文件
cat log.txt | tail -100         # 看日志最后 100 行
ps aux | grep python            # 找 python 进程
```

重定向 `>` `>>` 把输出存到文件：
```bash
echo "hello" > file.txt         # 覆盖
echo "hello" >> file.txt        # 追加
```

## 权限相关

### chmod——改权限

```bash
chmod 755 file                  # 常用权限
chmod +x script.sh              # 加执行权限
```

### chown——改所有者

```bash
chown user:group file
```

## 进程相关

### ps——看进程

```bash
ps aux                          # 所有进程
ps aux | grep python            # 找 python 进程
```

### kill——杀进程

```bash
kill PID                        # 正常退出
kill -9 PID                     # 强制杀死
```

### top/htop——看系统资源

```bash
top                             # 动态查看
htop                            # 更友好的界面（如果有的话）
```

比赛现场如果发现程序卡死，先看看 CPU 占用和内存占用。

## 网络相关

### ping——测试网络

```bash
ping 192.168.1.1
```

### curl——发 HTTP 请求

```bash
curl http://api.example.com
curl -X POST -d '{"key":"value"}' url
```

### ifconfig/ip——看网卡

```bash
ifconfig
ip addr
```

## 解压/压缩

```bash
tar -czvf archive.tar.gz dir/   # 压缩
tar -xzvf archive.tar.gz       # 解压
zip -r archive.zip dir/        # zip 压缩
unzip archive.zip              # zip 解压
```

## 常用快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl + C` | 取消当前命令 |
| `Ctrl + Z` | 暂停当前命令 |
| `Ctrl + L` | 清屏 |
| `Ctrl + A` | 跳到行首 |
| `Ctrl + E` | 跳到行尾 |
| `Ctrl + R` | 搜索历史命令 |
| `Tab` | 自动补全 |
| `!!` | 重复上一条命令 |

## 总结

这些命令不需要背下来，用多了自然就记住了。

我的建议是：先把 `ls`, `cd`, `cp`, `mv`, `rm`, `cat`, `less`, `tail`, `grep`, `find`, `ps`, `kill` 这十几个命令练熟，够你应付 90% 的场景了。

遇到不会的，现查都来得及。
