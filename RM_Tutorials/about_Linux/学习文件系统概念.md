# 学习文件系统概念

> 第一次用 Linux 的时候，我对着终端发了半天呆——这破系统，文件都跑哪去了？

其实 Linux 的文件系统没那么神秘，这篇帮你把这件事搞清楚。

## 一切皆文件

在 Linux 里，**所有东西都是文件**。

- 你的代码是文件
- 你的图片是文件
- 你的键盘是文件（设备文件）
- 你的硬盘是文件

甚至连进程，在 `/proc` 里也是以文件形式存在的。这种"一切皆文件"的哲学，让 Linux 的操作变得极其统一——你用同样的命令，可以读取数据、控制硬件、管理进程。

## 目录结构——从根开始

Linux 的目录树从 `/` 开始，叫根目录。

```
/
├── bin      # 放系统命令的
├── etc      # 配置文件
├── home     # 普通用户的家目录
├── root     # 超级用户的家目录
├── usr      # 安装的程序
├── dev      # 设备文件
├── proc     # 进程信息
├── tmp      # 临时文件
└── var      # 经常变化的文件，比如日志
```

刚开始不需要背下来，用多了自然就记住了。

## 绝对路径 vs 相对路径

**绝对路径**：从根目录开始的完整路径
```bash
/home/meko/project/src/main.cpp
```

**相对路径**：从当前目录开始的路径
```bash
./src/main.cpp      # 当前目录下的 src
../../meko          # 上两级目录
```

> 小技巧：写脚本的时候，尽量用相对路径。这样把整个项目拷贝到别的机器上也能跑。

## 常见目录的用途

| 目录 | 用来放什么 |
|------|-----------|
| `/home/用户名` | 你的个人文件 |
| `/opt` | 大型软件，比如 Python、OpenCV |
| `/usr/local` | 手动编译安装的软件 |
| `/tmp` | 临时文件，关机就清空 |

## 软链接和硬链接

这个概念容易把人搞晕：

- **硬链接**：同一个文件，两个名字。删一个不影响另一个。
- **软链接**：类似 Windows 的快捷方式，本质上是个指针。

```bash
# 创建软链接
ln -s 目标 链接名

# 创建硬链接
ln 目标 链接名
```

实际开发中，软链接用得更多。比如你的代码在 `/home/meko/project`，但你想在 `/workspace` 也能访问——建个软链接就行。

## 挂载——把分区变成目录

如果你插了个 U 盘，Linux 不会像 Windows 那样给你分配一个 E 盘、F 盘。它会把 U 盘"挂载"到某个目录下。

```bash
# 手动挂载
mount /dev/sdb1 /mnt/usb

# 卸载
umount /mnt/usb
```

比赛现场如果需要从 U 盘拷贝数据，这个命令早晚得用。

## 文件类型

用 `ls -la` 的时候，第一列会显示文件类型：

| 字符 | 类型 |
|------|------|
| `-` | 普通文件 |
| `d` | 目录 |
| `l` | 软链接 |
| `c` | 字符设备（键盘、终端） |
| `b` | 块设备（硬盘、U盘） |
| `p` | 管道 |
| `s` | socket |

## 总结

文件系统这事儿，说白了就是：
- Linux 一切皆文件
- 目录树从 `/` 开始
- 用绝对路径或相对路径定位文件
- 软链接是个很实用的东西

知道这些，足够了。剩下的，实践中慢慢就懂了。
