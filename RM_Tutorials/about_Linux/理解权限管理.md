# 理解权限管理

> 有次我在服务器上新建了一个文件，结果队友打不开。他当时看我的眼神，仿佛在说：你是不是故意的？

后来才知道，是权限没设置对。这事儿不复杂，但很重要。

## 基本概念

Linux 的权限模型由三个角色组成：

| 角色 | 含义 |
|------|------|
| `u` (owner) | 文件所有者 |
| `g` (group) | 文件所属组 |
| `o` (others) | 其他所有人 |

每个角色有三种权限：

| 权限 | 含义 | 数字 |
|------|------|------|
| `r` (read) | 读取 | 4 |
| `w` (write) | 写入 | 2 |
| `x` (execute) | 执行 | 1 |

## 怎么看权限——ls -l

```bash
$ ls -l file.txt
-rw-r--r-- 1 meko  meko  1024 Jan 15 10:30 file.txt
```

拆开看：

```
-         rw-         r--         r--
文件类型   所有者权限   组权限      其他人权限
```

- `-` 是普通文件（`d` 是目录，`l` 是软链接）
- `rw-` = 所有者有读写权限，没有执行权限
- `r--` = 组只有读权限
- `r--` = 其他人只有读权限

## 怎么改权限——chmod

### 数字法（最常用）

```bash
chmod 755 script.sh
```

| 数字 | 权限 |
|------|------|
| 7 | rwx = 4+2+1 |
| 6 | rw- = 4+2 |
| 5 | r-x = 4+1 |
| 4 | r-- |
| 0 | --- |

所以 `755` 的意思是：
- 所有者：7 (rwx)
- 组：5 (r-x)
- 其他：5 (r-x)

### 符号法

```bash
chmod u+x script.sh      # 给所有者加执行权限
chmod g-w file.txt       # 给组去掉写权限
chmod o=r file.txt       # 给其他人设置只读
chmod a+x file.sh        # a = all，给所有人加执行权限
```

## 怎么改所有者和组——chown

```bash
chown meko file.txt              # 改所有者
chown meko:robot file.txt        # 改所有者和组
chown :robot file.txt           # 只改组
```

> 只有 root 或者文件所有者才能改权限。

## 目录的权限

目录的权限和文件有点不一样：

| 权限 | 对目录的含义 |
|------|-------------|
| r | 能列出目录里的文件（ls） |
| w | 能在目录里创建/删除文件 |
| x | 能进入目录（cd） |

所以，如果一个目录没有 x 权限，你连 cd 都进不去，更别读写里面的文件了。

## 实际开发中的常见问题

### 问题 1：队友打不开我创建的文件

```bash
# 方案 1：改成 666（所有人都能读写）
chmod 666 file.txt

# 方案 2：把文件所在目录的组设置对
chgrp robot project/
chmod 770 project/
```

### 问题 2：脚本跑不起来

```bash
# 缺少执行权限
chmod +x run.sh
./run.sh
```

### 问题 3：Python 脚本 import 报错

有时候是权限问题，有时候是目录没加执行权限。

### 问题 4：SSH 密钥权限不对

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub
```

SSH 对权限要求很严格，差一点都不行。

## sudo——临时获得 root 权限

```bash
sudo command
```

需要输入密码，但不会显示。输入完回车就行。

> 比赛现场的机器通常只有你自己的账号，配置环境的时候经常需要 sudo。

## umask——新建文件的默认权限

你新建一个文件，默认权限是多少？

```bash
$ umask
0022
```

umask 是"掩码"，默认权限 666 减去 umask，就是新建文件的权限。

- 文件默认：666 - 022 = 644 (rw-r--r--)
- 目录默认：777 - 022 = 755 (rwxr-xr-x)

想改的话，`umask 027` 这样设置。

## 特殊权限——suid/sgid/sticky

偶尔会看到这种权限：

```bash
-rwsr-xr-x 1 root root  1234 Jan 15 10:30 script
```

那个 `s` 就是特殊权限：
- `suid`：运行的时候以文件所有者身份运行
- `sgid`：运行的时候以文件所属组身份运行
- `sticky`（在目录上）：只能删除自己的文件

一般情况用不到，了解就行。

## 实战建议

1. **项目目录设个组**：你们队几个人都加到一个组里，目录权限设为 770，文件默认 664，大家都能互相访问。

2. **重要文件设只读**：比如配置文件，chmod 444 防止误改。

3. **脚本加上执行权限**：chmod +x 是个常用操作。

4. **不要随便给 777**：除非真的需要。给得越多，风险越大。

## 总结

权限这事儿，核心就三点：
- 谁（owner/group/others）
- 能做什么（r/w/x）
- 怎么改（chmod/chown）

知道这些，够用了。
