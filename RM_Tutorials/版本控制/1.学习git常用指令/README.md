# 学习 Git 常用指令

刚接触版本控制的时候，我相信很多人和我一样——装好了 Git，配好了 SSH，打开终端准备大干一场，然后对着满屏的英文命令发懵。

那种感觉挺奇妙的。你知道这东西能帮你找回误删的代码，知道多人协作全靠它，但就是不知道从哪下手。

所以我想把入门阶段最常用的几个命令理一理。不是为了背命令，而是理解**什么时候该用什么**。

---

## git init —— 故事的开始

这个命令的意思是"把当前这个文件夹变成 Git 可以管理的仓库"。

```bash
git init
```

跑完之后你会发现多了一个 `.git` 的隐藏文件夹。恭喜你，你的文件夹现在有"记忆"了。

> 小提示：别手贱去动 `.git` 里面的东西。那里存着所有的版本历史，改坏了就只能祈祷了。

---

## git clone —— 拿来别人的项目

如果你想参与一个已经存在的项目，比如你链接里的那个 RM-Vision-Learning：

```bash
git clone https://github.com/MEKO747/RM-Vision-Learning.git
```

这会在当前目录创建一个同名文件夹，把整个仓库复制下来。

有时候仓库比较大，clone 下来要好一会儿。那种时候我会去喝杯水，回来差不多就好了。

---

## git status —— 你的代码现在什么情况

这是你每天要敲无数次的一个命令。

```bash
git status
```

它会告诉你：
- 哪些文件被修改了但还没暂存
- 哪些文件刚被暂存准备提交
- 当前在哪个分支

我刚学的时候不习惯看这个输出，觉得全是英文。后来强迫自己每天 `status` 三遍，两天就习惯了。

---

## git add 和 git commit —— 存档你的进度

这是最核心的一对操作。

**git add** 把文件从"工作区"移到"暂存区"：
```bash
git add filename.c      # 暂存单个文件
git add .               # 暂存所有改动的文件
```

**git commit** 把暂存区的内容生成一个快照：
```bash
git commit -m "修复了视觉算法的边界检测问题"
```

-m 后面跟的那句话就是版本说明，尽量写清楚这次改了什么。三年后回来看 commit 历史，你就知道这条规矩有多重要了。

---

## git push —— 把本地进度上传

commit 只是在本地记录了版本。想让远程仓库（GitHub）也同步这些记录：

```bash
git push origin main
```

origin 是远程仓库的默认名字，main 是分支名。如果不加参数，Git 会用默认配置推送。

第一次 push 的时候，GitHub 会让你验证身份。配好 SSH Key 之后就没这事了，体验会好很多。

---

## git pull —— 把别人的更新拉下来

团队协作的时候，别人可能在你写代码的同时也在往 main 分支提交代码。

```bash
git pull origin main
```

这会把远程的最新内容拉下来并合并到你的当前分支。

我之前踩过一个坑：没 pull 就直接 push，结果本地和远程冲突了，花了半小时才解决。所以每次开始写代码前先 pull 一把，是好习惯。

---

## git log —— 查看历史

想知道这个项目都经历了什么：

```bash
git log --oneline
```

每行一个 commit，简短信息加哈希值，一目了然。

```bash
git log --graph --oneline --all
```

加上 `--graph`，还能看到分支和合并的可视化图形。

---

## 几个常见问题

**Q: 提交信息写错了怎么办？**
```bash
git commit --amend
```
这会修改最后一次 commit 的信息。但注意，这条命令会改变 commit 的哈希值，如果已经 push 到远程，就别再 amend 了。

**Q: 暂存错了文件怎么办？**
```bash
git reset HEAD filename
```
这可以把一个文件从暂存区移出来，但保留修改内容。

**Q: 怎么忽略一些文件不被跟踪？**
在项目根目录创建 `.gitignore` 文件，写入不想被跟踪的文件名或规则，比如 `*.o`、`build/` 之类的编译产物。
